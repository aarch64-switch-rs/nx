pub fn new_stray( gpu_ctx: Arc<RwLock<Context>>, surface_name: Option<&'static str>, buffer_count: u32, block_height: BlockLinearHeights, ) -> Result<Self>

Creates a new managed layer (application/applet window) that can be drawn on overlay elements. These layers exist on top of stray layers and application/applet UIs, and can be Z-order vertically layered over each other. The GPU provides the alpha blending for all layers based on the committed frame.

pub fn render<T>( &mut self, clear_color: Option<ColorFormat>, runner: impl Fn(&mut BufferedCanvas<'_, ColorFormat>) -> Result<T>, ) -> Result<T>

Check out a canvas/framebuffer to draw a new frame. This frame is buffered to a linear buffer before being swizzled into the backing buffer during frame-commit. This provides better performance for line-based renderers as writes will be sequential in memory. There are cache misses during commit, but that is still better performance than mapped page churn for GPU-mapped memory.

Source

pub fn render_unbuffered<T>( &mut self, clear_color: Option<ColorFormat>, runner: impl Fn(&mut UnbufferedCanvas<'_, ColorFormat>) -> Result<T>, ) -> Result<T>

Check out a canvas/framebuffer to draw a new frame, without a linear buffer. This can be used in memory constrained environments such as sysmodules, but also provides slightly better performance for frames that draw in block rather than scan-lines (e.g. JPEG decoding).
